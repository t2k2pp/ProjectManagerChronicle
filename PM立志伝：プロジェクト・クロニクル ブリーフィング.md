# PM立志伝：プロジェクト・クロニクル ブリーフィング

## エグゼクティブサマリー

本ブリーフィングは、プロジェクトマネジメント（PM）体感シミュレーションゲーム『PM立志伝：プロジェクト・クロニクル』の全体像を統合的に解説するものである。本作は、カイロソフト社の『ゲーム発展国++』が持つ中毒性のある経営・育成ループと、コーエーテクモゲームス社の『太閤立志伝V』が持つ自由度の高いキャリア・スキル習得システムを融合させ、PMの実務を体感的に学習することを目的とした、全く新しい学習アプリケーションである。

**勝利の条件は「運」ではない。「正しい知識」と「判断」だ。** 本作は運要素を意図的に排除し、PMBOK（プロジェクトマネジメント知識体系）に準拠した論理的な意思決定が成功の唯一の鍵となるよう設計されている。プレイヤーはデスマーチを回避し、伝説のPMを目指す過程で、教科書では得られない生きたPMスキルを習得する。

主要な特徴は以下の通りである：

- **ダブル・ヘキサゴン・システム:** キャラクターの能力を、業務スキルを示す「表の六角形」と、人間性や適性を示す「裏の六-角形」の合計12項目で表現。これにより、多面的でリアルな人物像と、奥深いチームビルディングの戦略性を実現する。
- **交渉カードバトル:** 「スコープ・クリープ」や「トレードオフ」といったPM用語が「武器」となるカードバトルシステムを導入。ステークホルダーとの駆け引きを戦略的に体験できる。
- **AIによるフィードバック:** プロジェクト完了時に生成AIが全行動ログを解析し、個別に最適化された「プロジェクト完了報告書」を生成。自らの行動を客観的に振り返り、失敗を「最強の資産」に変える学習サイクルを確立する。

本作は、未来のPM、現役のリーダー、そしてマネジメントに関心のある全てのビジネスパーソンをターゲットとし、安全な環境で実践的な経験を積む機会を提供する。

## 1. コアコンセプトと設計思想

『PM立志伝』は、単なるシミュレーションゲームの枠を超え、次世代のPMを育成する「デジタル道場」として構想されている。その設計は、2つの名作ゲームのDNAと、現実のPM理論に基づいている。

### 1.1 設計の二本柱：名作のDNAの融合

本作のコンセプトは、以下の2つのゲームが持つ面白さの本質を掛け合わせることから生まれた。

|   |   |   |
|---|---|---|
|特徴|ゲーム発展国++ (カイロソフト)|太閤立志伝V (コーエーテクモゲームス)|
|**主役**|会社（組織）|個人（主人公）|
|**コア体験**|経営・開発のループ（組織運営）|人生のロールプレイング（個人の成長物語）|

この融合により、**組織を運営するマクロな視点**（チーム編成、進捗管理、QCDのバランス取り）と、**PM個人として成長するミクロな視点**（キャリアパス、スキル習得、交渉術）を両立させた、深みのあるゲーム体験を創出する。

### 1.2 コア原則：PMBOK準拠の論理的思考

**"勝利の条件は、運ではなく「PMBOK（プロジェクトマネジメント知識体系）に準拠した正しい判断」です。"**

ゲーム内のイベントや選択肢は、すべてPMBOKのフレームワークに基づいて設計されている。ランダムに見えるトラブルも、計画段階での「リスク登録簿」への記載有無でダメージが変動するなど、論理的な因果関係が設定されている。プレイヤーは、行き当たりばったりのプレイではなく、常に先を見越した管理と判断を求められる。

## 2. 革新的なキャラクターシステム

プロジェクトの成否は「人」で決まる。本作では、単なる労働力ではない、人間味あふれるキャラクターたちがゲームの中核をなす。

### 2.1 能力値『ダブル・ヘキサゴン』

キャラクターの能力は、業務スキルを示す「表の六角形」と、人間性や適性を示す「裏の六角形」の合計12項目で表現される。この二層構造は、「一見役に立たないような人が、実はチームの危機を救う」という現実の職場における力学を再現する。

|   |   |   |
|---|---|---|
|能力群|パラメータ|主な役割|
|**【表】プロジェクトスキル**<br>(青の六角形)|**設計**|手戻り発生率の抑制|
||**製造**|基本的な進捗スピード|
||**評価**|品質スコアとバグ発見率|
||**折衝**|外部からの要求に対する防御力|
||**提案**|状況を打開するブレイクスルーの発生|
||**判断**|タスクの優先順位付けと自律行動|
|**【裏】チーム運営力**<br>(赤の六角形)|**庶務**|PMの管理コスト削減|
||**幹事**|チームの士気回復と団結力向上|
||**給仕**|メンバーの体力維持|
||**雑談**|メンバー間の対立イベント防止|
||**美貌**|周囲のメンバーへの能力ボーナス|
||**幸運**|低確率で危機を救うラッキーイベントの発生|

### 2.2 人材の多様性と成長

- **固有NPCと『二つ名』システム:** 『太閤立志伝V』の有名武将にヒントを得て、特別な能力を持つ固有NPCが登場する。当初は扱いにくい人物に見えるが、信頼関係を築くことで真の能力が解放され、**『遺産を守る者』**や**『深夜のコミット怪人』**といった「二つ名」と共にプロジェクトの切り札となる。
- **一般キャラクターのランダム性と『覚醒』:** ランダム生成される一般キャラクターにも「覚醒タイプ」が秘められている。ステレオタイプを排除した完全ランダムな能力配分により、プレイごとに異なる「埋もれた原石」を発掘・育成する楽しみを提供する。
- **成長ルール:** 【表】のスキルは経験によって成長するが、【裏】のスキルは原則として成長しない。これにより、「スキルは教育できるが、人間性は採用で見抜く必要がある」という現実的な採用戦略の重要性が生まれる。

### 2.3 リアルな体力管理

キャラクターの体力は**「最大体力（タンクの大きさ）」**と**「回復力（回復スピード）」**の2軸で定義され、以下のタイプが存在する。

- **瞬発型（若手タイプ）:** 最大体力は低いが、回復力が高い。短期決戦向き。
- **持続型（ベテランタイプ）:** 最大体力は高いが、回復力が低い。一度枯渇すると復帰に時間がかかる。

PMがメンバーを酷使し、APが一定以下になると**「蓄積疲労」**が発生し、最大体力が減少する。これにより、「体力がある人ほど使い潰すと再起不能になる」というマネジメントの怖さを表現している。

## 3. PM業務を再現する主要メカニクス

ゲームシステムは、プロジェクトマネージャーが日常的に直面する課題と意思決定のサイクルを忠実に再現するよう設計されている。

### 3.1 週次PDCAゲームサイクル

ゲームは「1ターン＝1週間」で進行し、マネジメントの本来のリズムを再現する。

1. **Monday Plan（週初めの計画）:** プレイヤーはガントチャート上でタスクへのアサインを行い、「品質第一」「突貫工事」といった方針カードを選択して週の方針を決定する。
2. **Mid-Week Event（平日の実行とイベント）:** 計画に基づき自動で進行。この間に「駆け込み要求」や「仕様の曖昧さ発見」といったイベントが発生し、プレイヤーの判断が求められる。
3. **Friday Review（週末の振り返り）:** 週報が表示され、**EVM（アーンド・バリュー・マネジメント）**指標である**SPI（スケジュール効率指数）**と**CPI（コスト効率指数）**によって成果を客観的に確認し、次週の戦略を練る。

### 3.2 交渉カードバトル

顧客や上司との重要な対話は、戦略的なカードバトルで表現される。

- **PM用語が武器に:** 「スコープ・クリープ」（防御）、「トレードオフ」（交渉）、「エスカレーション」（特殊）といったPM用語が効果を持つカードとして登場する。
- **『直談判』カード:** ここぞという場面で使える特殊カード。プレイヤーが自由記述で入力した説得内容をAIが評価し、その成否がバトルの結果に反映される。「言語化能力」が試される。

### 3.3 カレンダーと季節イベント

ゲームは4月開始の12ヶ月制で進行し、現実の季節が戦略的な制約となる。

- **三大「鬼門」イベント:** ゴールデンウィーク、夏季休暇、年末年始は、チームの稼働率が低下する固定イベントとして配置される。
- **『休暇調整パズル』:** 休暇前には、メンバーごとの休暇スタンス（バカンス勢、出稼ぎ勢など）を考慮し、予算や信頼度をコストとして支払いながらシフトを調整するミニゲームが発生。「敵はバグにあらず、人の心にあり」というPMの苦悩を再現する。

### 3.4 ワールドジェネレーションと主人公設定

- **初期環境の固定生成:** ゲーム開始時に自社、競合他社、市場案件が固定生成される。「後出しジャンケン」のない公平な環境で、プレイヤーの先見性と戦略的計画が試される。
- **主人公メイキング:** 「年齢設定（20代/30-40代/50代+）」で基礎ステータスと体力タイプが決まり、その後の「PMタイプ診断」で得意分野（エンジニア上がり、営業上がり等）が決定する。これにより、プレイヤーごとに異なるプレイスタイルが生まれる。

## 4. 学習効果を最大化する機能

本作はエンターテインメントであると同時に、効果的な学習ツールとして設計されている。

### 4.1 AIによるパーソナライズされたフィードバック

本作の最も革新的な機能が、生成AIを活用した**「プロジェクト完了報告書」**である。プロジェクト完了時、全プレイログをAIが解析し、「ベテランPMメンター」のペルソナで詳細なフィードバックを提供する。

|   |   |
|---|---|
|レポート項目|内容|
|**【Your PM Style】**|プレイヤーの意思決定傾向から「バランス調整型」「リスク許容型」といったPMスタイルを分析・言語化する。|
|**【Critical Decisions】**|プロジェクトの成否を分けた「運命の分岐点」を特定し、なぜその判断が正解（または不正解）だったのかをPMBOKの理論を交えて解説する。|
|**【Vocabulary Note】**|ゲーム中で遭遇した専門用語を、プレイヤー自身の行動と紐付けてリスト化する。|
|**【KPT】**|Keep（良かったこと）、Problem（悪かったこと）、Try（次に試すべきこと）のフレームワークで、具体的な改善アクションを提示する。|

この「Play → Speak → Review → Learn」のサイクルにより、失敗すらも「最強の資産」となり、知識が確かなスキルへと昇華される。

### 4.2 直感的なUI設計『PMコックピット』

メイン画面は、PMが日々向き合う多様な情報を一元管理する「戦況図」をコンセプトに設計されている。

- **プロジェクトHUD（上部）:** カレンダー、EVMメーター（SPI/CPI）、残り予算など、プロジェクトの健康状態を示す。
- **オフィスビュー（中央左）:** メンバーの体力や状態を座席表のようなビジュアルで直感的に把握できる。
- **ガントチャート（中央右）:** 計画線と実績線の乖離が一目でわかる進捗管理の中核。
- **アクションメニュー（下部）:** アサインや方針指示など、プレイヤーが操作を行うコマンドデッキ。

このUIは、プレイヤーが情報検索に忙殺されることなく、迅速かつ質の高い「判断」に集中できるよう支援する。

## 5. 主要な専門用語とゲーム内での役割

本作を通じて、以下の主要なプロジェクトマネジメント用語を実践的に学ぶことができる。

|   |   |
|---|---|
|用語|ゲーム内での役割・学習方法|
|**PMBOK**|ゲーム全体の成功ロジックの根幹。PMBOKに準拠した判断が常に正解となる。|
|**ステークホルダー**|個性的な性格を持つNPCとして登場。彼らの期待を調整し、交渉することが重要となる。|
|**QCD**|プロジェクト完了時の最終スコアを決定する3大指標。トレードオフの関係性を常に意識させられる。|
|**WBS**|計画フェーズで、パズル形式のミニゲームとしてタスクを分解・構築する。|
|**クリティカルパス**|WBS作成時に特定が求められる最重要タスク経路。ここにエースを配置することが攻略の鍵となる。|
|**リスクマネジメント**|計画時に「リスク登録簿」に備えを記載することで、トラブル発生時のダメージを軽減できる。|
|**EVM**|メイン画面のメーターで常に表示され、SPI/CPIを通じてプロジェクトの健康状態を客観的に把握する訓練となる。|
|**スコープ・クリープ**|交渉カードバトルで敵の攻撃として発生。防御カードで撃退することで、断る重要性を学ぶ。|
|**KPT**|AIによる完了報告書のフレームワーク。経験を次の成功に繋げる思考法を身につける。|
