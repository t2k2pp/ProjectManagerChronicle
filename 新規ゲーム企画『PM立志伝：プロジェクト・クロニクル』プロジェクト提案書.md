# 新規ゲーム企画『PM立志伝：プロジェクト・クロニクル』プロジェクト提案書

本書は、投資家および開発チームを対象とした公式な企画提案書です。専門的かつ説得力のあるトーンを維持し、プロジェクトのビジョン、市場価値、そして実現可能性を明確に伝えます。

--------------------------------------------------------------------------------

## 1. 企画概要 (Project Overview)

本企画が提案するのは、単なるエンターテインメントではありません。これは、現代ビジネス必須の専門スキルであるプロジェクトマネジメント（PM）を体得させるための、次世代の企業研修を定義する「デジタル道場」です。プレイヤーは安全なシミュレーション環境で成功と失敗の体験を高速で繰り返し、教科書では得られない「生きたPMスキル」を習得します。本作は、従来の受動的な研修に代わる、実践的かつ効果的なB2Bトレーニングソリューションとして、業界の新たな標準を打ち立てることを目指します。

- **タイトル案:** 『PM立志伝：プロジェクト・クロニクル』
- **キャッチコピー:** ～デスマーチを回避し、伝説のPMを目指せ～
- **ジャンル:** プロジェクトマネジement体感シミュレーション / 学習アプリ
- **コンセプト:** カイロソフト社の名作『ゲーム発展国++』が持つ、中毒性の高い組織運営・育成ループの面白さと、コーエーテクモゲームス社の傑作『太閤立志伝V』が持つ、個人の成長と交渉を軸としたロールプレイングの奥深さを融合させます。これにより、プレイヤーは組織を動かすマクロな視点と、PM個人のキャリアを磨くミクロな視点の両方から、プロジェクトマネジメントの実務を体感的に学習します。
- **コア原則:** 本作の勝利条件は「運」ではありません。プロジェクトの成否は、世界標準の知識体系である「PMBOK（プロジェクトマネジメント知識体系）」に準拠した、プレイヤーの**正しい知識と論理的な判断**によってのみ決定されます。これにより、再現性が高く、学習効果に直結するゲーム体験を提供します。

本企画は、純粋なエンターテインメント性と実用的な教育価値を両立させることで、新たな市場を切り拓く可能性を秘めています。次章では、本企画が狙う巨大な市場機会と、その中核をなすターゲット層について詳述します。

## 2. 市場背景とターゲット層 (Market Context and Target Audience)

現代ビジネスにおいて、プロジェクトマネジメント能力は不可欠なスキルであり、その市場規模は数十億ドルに達します。しかし、既存の企業研修市場は、知識を一方的に伝達する座学やビデオコースといった「受動的な学習教材」で飽和状態にあり、実践的な判断力を養うという点で決定的なギャップが存在します。本企画は、この巨大かつ満たされていない需要、すなわち「失敗が許容される環境で、結果が伴うリアルな経験を通じてスキルを鍛えたい」というニーズに真正面から応えるものです。これは、体験型学習プラットフォームによる市場破壊の絶好の機会です。

|   |   |   |
|---|---|---|
|ターゲット層|特徴|本ゲームから得られる価値|
|**1. 未来のPM層（学生・若手社会人）**|これからPMやリーダーを目指す、あるいはキャリアに漠然とした不安を抱える層。実務経験が少ない。|プロジェクト全体の流れと、PMが日々直面する課題を疑似体験できる。専門用語や思考プロセスを、キャリアの早い段階で楽しく学べる。|
|**2. 現役のPM・リーダー層**|すでにPMとして活動しているが、自己流のマネジメントに限界を感じている、あるいはより大規模な案件に挑戦したいと考えている層。|自身のマネジメントスタイルを客観的に見直し、PMBOKに基づいた知識との答え合わせができる。現実では試せない大胆な戦略をシミュレーションできる。|
|**3. マネジメントに関心のある一般ビジネスパーソン**|PM職ではないが、チームでの業務遂行やリーダーシップに関心を持つ全てのビジネスパーソン。|「なぜプロジェクトは炎上するのか」「なぜ上司はあのような判断をするのか」といった、組織力学の裏側を理解できる。自身の業務調整能力向上に繋がる。|

これら全ての層に共通する「体験を通じた学習意欲」こそが、本企画の成功を支える強固な基盤です。それでは次に、彼らの知的好奇心を満たすための具体的なゲームメカニクスについて解説します。

## 3. コアコンセプトと主要メカニクス (Core Concept and Key Mechanics)

本ゲームの根幹をなす設計思想は、PMが直面する最大の挑戦、すなわち「混沌と制約の中で価値を創造する」というダイナミックな戦略シミュレーションを提供することにあります。これから解説する各メカニクスは、それぞれが独立して存在するのではなく、相互に作用し、プレイヤーに常にトレードオフの決断を迫るよう緻密に連携しています。

### 3.1. ゲームサイクル：1ターン＝1週間のPDCA

ゲーム進行の基本単位は「1ターン＝1週間」とし、現実のマネジメントにおける週次定例やアジャイル開発のスプリントのリズムを再現します。これにより、プレイヤーは日々のマイクロマネジメントではなく、週次の傾向を掴んで次の一手を考えるという、PM本来の思考を追体験できます。

1. **週初めの計画（Monday Plan）** 今週の目標達成に向け、リソース配分と方針を決定する戦略フェーズです。プレイヤーはガントチャートを見ながらタスクへのアサインを決定し、さらに「品質第一」や「突貫工事」といった方針カードを選択することで、チーム全体の作業効率やリスクに影響を与えます。
2. **平日の実行とイベント（Mid-Week Event）** 計画を確定すると自動で進行し、計画に基づき予期せぬ出来事への対応が求められます。この間、プレイヤーのアサインやメンバーの能力に応じて、トラブルや好機といった様々なイベントが発生し、プレイヤーはその都度、判断を求められます。
3. **週末の振り返り（Friday Review）** 金曜日の夕方には週次レポートが表示されます。EVM（アーンド・バリュー・マネジメント）指標である**SPI（スケジュール効率指数）**と**CPI（コスト効率指数）**によって、計画に対する進捗とコストの状況が可視化されます。この結果を分析し、次週の計画へと活かします。

### 3.2. キャラクターシステム：個性の集合体が織りなすドラマ

本ゲームの最大の特徴は、単なる「労働力」ではない、人間味あふれるキャラクターたちです。彼らの能力、性格、そして価値観がプロジェクトの成否を大きく左右します。

- **能力値『ダブル・ヘキサゴン』** キャラクターの能力は、業務スキルを示す「**表スキル (Professional Skills)**」と、人間性や適性を示す「**裏スキル (Human Skills)**」の合計12項目で表現されます。これにより、「技術は超一流だがチームを破壊する天才」や「スキルは低いがムードメーカーとして不可欠な潤滑油」といった、多面的でリアルな人物像を描き出します。
    - **表スキル (Professional Skills)**
        - 設計
        - 製造
        - 評価
        - 折衝
        - 提案
        - 判断
    - **裏スキル (Human Skills)**
        - 庶務
        - 幹事
        - 給仕
        - 雑談
        - 美貌
        - 幸運
- **固有NPCと『二つ名』システム** 物語には、『深夜のコミット怪人』や『微笑みの仕様粉砕者』といった、一癖も二癖もある固有NPCが登場します。プレイヤーがPMとして彼らの本質を見抜き、信頼関係を築くことで真の能力が解放され、プロジェクトの切り札となる「二つ名」が開示されます。
- **一般キャラクターのランダム性と『覚醒』** ランダム生成される一般キャラクターにも、「**二つ名持ち**（レアスキルを持つ）」「**実務の鬼**（二つ名はないがステータスが急上昇）」「**一芸特化**（特定分野で輝く）」といった「覚醒タイプ」が秘められています。プレイヤーは「埋もれた原石」を見つけ出し、育成するというリプレイ性の高い人材発掘の楽しみを味わえます。
- **体力システム『瞬発型と持続型』** 体力は「最大体力（タンクの大きさ）」と「回復力（回復スピード）」の2軸で定義されます。これにより、「無理は効かないが回復が早い若手（瞬発型）」と、「連続稼働はできるが一度倒れると復帰が遅いベテラン（持続型）」という運用特性が生まれます。PMには、メンバーを使い潰さないための高度なリソース管理能力が求められます。

最終的に、このキャラクターシステムはPMを多角的な方程式の解決へと導きます。すなわち、『ダブル・ヘキサゴン』が示すポテンシャルを最大限に引き出し、『二つ名』と『覚醒』を通じて才能を発掘・育成し、同時に『体力システム』が課す人間的な制約を管理する。この複雑なパズルを解き明かすことこそ、現実のリーダーシップにおける核心的な挑戦の再現に他なりません。

### 3.3. 交渉カードバトル：PM用語が武器になる

ステークホルダー（顧客、上司など）との重要な対話は、カードバトル形式で表現されます。「スコープ・クリープ」「トレードオフ」「エスカレーション」といったPMBOKの専門用語が、攻撃、防御、特殊効果を持つカードとして登場します。これにより、プレイヤーは楽しみながら、自然と専門用語とその実践的な使い方を習得できます。

### 3.4. カレンダーシステム：季節という名の不可避な制約

ゲームは4月から始まる12ヶ月制で進行し、ゴールデンウィーク・夏季休暇・年末年始といった長期休暇は、プロジェクトの進捗を阻む「鬼門イベント」として固定配置されています。キャラクターごとに設定された『休暇スタンス』を考慮し、休暇前に『シフト調整パズル』を解かなければなりません。プレイヤーは「**休日割増手当**」で予算を使い稼働を確保するか、「**代休手形**」で問題を先送りするか、あるいは「**PM自身が出勤**」して己を犠牲にするか、という苦渋の選択を迫られます。これは、PMが直面する現実的な人間関係調整の苦悩と面白さを再現します。

これら全てのメカニクスは、PMの本質的な挑戦を体験するための「実践（Play）」の場そのものです。PDCAサイクル、人材管理、交渉、そして予期せぬ制約への対応、これら全てが有機的に絡み合い、次章で詳述する学習サイクルを強力に駆動させるのです。

## 4. 学習要素と教育的価値 (Learning Elements and Educational Value)

本企画は単なるゲームに留まらず、プレイヤーが楽しみながら実践的なPMスキルを習得できる、効果的な学習ツールとして設計されています。ゲームをクリアする頃には、プレイヤーがPMとしてのマインドセットとスキルセットを自然に体得している状態を目指します。

本ゲームは、以下の4ステップから成る学習サイクルを高速で繰り返すことで、知識の定着を促します。

1. **Play (実践)** PMとして、プロジェクトの様々な局面で判断を下します。成功もあれば、手痛い失敗もあります。
2. **Speak (言語化)** 交渉カードバトルにおける『**直談判**』カードなど、重要な局面でプレイヤーは自分の言葉で相手を説得することを求められます。思考を言葉にする訓練は、学習内容を深く定着させます。
3. **Review (振り返り)** プロジェクト完了後、後述する生成AIによるレポートを通じて、自らの行動がPM理論においてどのような意味を持っていたのかを客観的に振り返ります。
4. **Learn (習得)** レポートで示された専門用語や改善点を深く理解し、次のプロジェクトで実践することで、学んだ知識を確かなスキルへと昇華させます。

### 生成AIによるパーソナライズされたフィードバック

本企画の最も革新的な機能が、生成AIを活用した「プロジェクト完了報告書」です。ゲーム中の全行動ログをAIが解析し、「ベテランPMメンター」の視点から、一人ひとりに最適化された詳細なフィードバックを提供します。

- **【Your PM Style】あなたの行動特性** プレイヤーの意思決定ログから、「リスク回避型」「コスト重視型」といったPMとしての行動傾向を分析・言語化します。これにより、自身の強みと弱みを客観的に把握できます。
- **【Critical Decisions】運命の分岐点** プロジェクトの成否を分けた重要な局面をピックアップし、プレイヤーの判断がなぜ正解（または不正解）だったのかを、PMBOKの理論を交えて具体的に解説します。
- **【Vocabulary Note】今回学んだ用語帳** ゲーム中で遭遇した「WBS」「クリティカルパス」といった専門用語が、プレイヤー自身の行動と紐付けられた形でリスト化され、生きた知識として記憶に残ります。
- **【KPT】次へのアクション** Keep（良かったこと）、Problem（悪かったこと）、Try（次に試すべきこと）のフレームワークで、具体的な改善アクションを提示し、次のプレイへの明確な目標を与えます。

この「体験学習ループ」こそが、従来の座学的な学習では決して得られない「生きたPMスキル」をプレイヤーに授けるのです。

## 5. ゲームフローとUI設計 (Game Flow and UI Design)

本ゲームが内包する複雑なシステムを、プレイヤーにストレスなく、かつ直感的に理解させるためには、洗練されたUI/UX設計が不可欠です。本企画では、PMが日々向き合う多様な情報を一元管理する「戦況図（コックピット）」をコンセプトに、視認性と操作性を両立させた画面設計を目指します。

### 5.1. ゲーム開始フロー：世界に一つだけの舞台設定

1. **主人公メイキング** 最初に、プレイヤー自身の分身となる主人公を作成します。「年齢設定」によって体力や初期スキルといった基礎ステータスが決定され、その後の「PMタイプ診断」を通じて得意分野がカスタマイズされ、プレイヤーごとに異なるプレイスタイルが生まれます。
2. **ワールドジェネレーション** ゲーム開始時に自社、競合他社、市場案件などのエコシステムを固定生成します。これにより、「後出しジャンケン」のような理不尽なランダム性が排除され、成功が予測不可能なイベントへの反応ではなく、先見性と戦略的計画によって決定される公平な競争環境が保証されます。

### 5.2. メイン画面『PMコックピット』：戦況の一覧化

ゲームプレイの大半を過ごすメイン画面は、以下の4つのエリアで構成され、プロジェクトの全体像を一目で把握できるよう設計されています。

- **上部: Project HUD** カレンダー（現在の週、季節）、**EVM Meters**（SPI/CPI）、残り予算など、プロジェクトの全体的な健康状態を示す計器類が表示されます。
- **中央左: Office View** メンバーの状態をオフィスの座席表のようなビジュアルで表現します。各キャラクターの頭上には体力ゲージが表示され、誰が疲弊しているかを直感的に把握できます。
- **中央右: Gantt Chart** プロジェクト進捗の心臓部です。計画線の上を実績線が伸びていくことで、計画との乖離が一目瞭然となります。遅延しているタスクは赤く発光し、プレイヤーに警告を発します。
- **下部: Action Menu** 週の初めにタスクをアサインしたり、チーム全体の方針を指示したりと、プレイヤーが具体的なアクションを行うためのエリアです。

このコックピット設計は、プレイヤーをデータ検索から解放し、迅速かつ質の高い「判断」を下すことに集中させることを可能にします。

## 6. 開発ロードマップ（案） (Proposed Development Roadmap)

リスクを最小化し、早期にコア・ループの有効性を検証するため、我々は段階的なMVP（実用最小限の製品）ファーストのアプローチを提案します。この戦略により、開発初期段階で強固な基盤を確立し、その上で二次的機能やコンテンツを計画的に追加していくことで、リソースの浪費を防ぎ、プロジェクトの成功確率を最大化します。

|   |   |   |
|---|---|---|
|フェーズ|目標|主な実装機能|
|**フェーズ1: MVP**|**コア・ループの証明:** 週次の計画・実行・振り返りのサイクルが、根本的に面白く、繰り返しプレイ可能であることを確立する。|・週次PDCAサイクルの基本ロジック<br>・ダブルヘキサゴンによるキャラクター能力値システム<br>・シンプルな交渉カードバトルとEVMによる評価システム<br>・固有NPCの「二つ名」は、信頼度によるステータス上昇のみ実装|
|**フェーズ2: フレーバーアップデート**|**没入感の深化:** 世界観を豊かにし、キャラクターへの愛着を醸成する演出を強化する。|・12ヶ月カレンダーと季節イベント（GW、夏季休暇など）の導入<br>・AIによる「プロジェクト完了報告書」のプロトタイプ実装<br>・固有NPCの「二つ名」解放時のカットイン演出やテキスト追加<br>・UIのビジュアルブラッシュアップ|
|**フェーズ3: イベントアップデート**|**戦略的深度の拡張:** 複雑なイベントとキャラクター固有能力を実装し、リプレイ性と高度な戦略性を提供する。|・入札バトルシステム、公的機関案件などの特殊プロジェクト<br>・キャラクターの特性に紐づいた固有イベントの実装<br>・『直談判』カードなどの自由記述システムの導入<br>・生成AIレポートの機能拡充と精度向上|

このロードマップに基づき、各フェーズで明確な目標を設定し、リスクを管理しながら着実にプロジェクトを推進することで、高品質な製品を計画通りに市場へ届けることが可能であると確信しています。

## 7. まとめ (Conclusion)

本企画『PM立志伝：プロジェクト・クロニクル』は、単なるシミュレーションゲームの枠を遥かに超え、次世代のPMを育成するための革新的な教育プラットフォームであり、企業研修の新たな業界標準を打ち立てるものです。

これは、エンターテインメントと教育が融合した「デジタル道場」です。プレイヤーは熱中する中で、現実のビジネスで通用する思考法、判断基準、そして人間関係構築スキルを体得していくでしょう。

我々にはビジョンと設計、そして市場機会があります。共にプロフェッショナル育成の未来を築き上げましょう。皆様の参加を心よりお待ちしております。
