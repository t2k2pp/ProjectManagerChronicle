# プロジェクトマネージャー体感ゲーム キャラクターパラメータ仕様書

--------------------------------------------------------------------------------

## 1.0 はじめに

本文書は、プロジェクトマネージャー体感ゲームの根幹をなすキャラクターの能力値（パラメータ）に関する詳細仕様を定義するものです。これは、ゲームの企画意図をシステムに落とし込み、エンジニアリングチームが実装を行う際の唯一の拠り所となる技術文書となります。

本ゲームのキャラクターは、プロジェクトを成功に導くために必要な能力を多角的に表現するため、**【表】のプロジェクトスキル**と**【裏】のチーム運営スキル**からなる、合計12のパラメータで構成されます。この「表裏12パラメータ」が、プレイヤーに現実のプロジェクトマネジメントさながらの奥深い人材活用と「やりくり」の体験を提供します。

## 2.0 パラメータの基本構造：ダブル・ヘキサゴン

キャラクターの能力構造には、**「ダブル・ヘキサゴン」**システムを採用します。このシステムは、古典的な経営シミュレーションに見られるプロジェクト単位のリソース管理（『ゲーム発展国』的要素）と、ロールプレイングゲームが持つ個人のスキルや交渉の奥深さ（『太閤立志伝V』的要素）を融合させるという、本作のコア・コンセプトを体現したものです。この構造は、プレイヤーに対して単なるスキル偏重ではない、多角的な人材マネジメントを促す戦略的な意図を持っています。

- **【表】プロジェクトスキル（青の六角形）:** プロジェクトの進捗、品質、外部との交渉といった、業務遂行に直接的に関わる能力群です。
- **【裏】チーム運営力（赤の六角形）:** チームの士気、人間関係、コスト管理、そして予測不能な「運」といった、プロジェクトの土台を支える間接的な能力群です。

UI上では、これら2つの能力群はそれぞれ青と赤のレーダーチャートとして表示され、キャラクターの得意・不得意が一目でわかるよう設計されます。次章からは、まずプロジェクトの成果に直結する【表】のパラメータについて詳述します。

## 3.0 【表】プロジェクトスキルパラメータ

このセクションでは、プロジェクトのQCD（品質・コスト・納期）に直接的な影響を与える6つの重要な能力値を定義します。いわば、プロジェクトにおけるキャラクターの**「攻撃力」**や**「防御力」**として機能し、プロジェクトの成否を大きく左右します。

### 3.1 設計 (Design)

プロジェクトの上流工程における品質担保の要であり、後の工程での致命的な手戻りを防ぐための最も重要な能力の一つです。（このパラメータは、プレイヤーに「急がば回れ」というPMの鉄則を体感させるためのものです。）

|   |   |
|---|---|
|項目|詳細|
|**定義**|プロジェクト進行中に発生する**「手戻り発生率」**を抑制する能力。|
|**ゲーム内効果**|- **高:** 「仕様漏れ」や「設計ミス」といった手戻りイベントの発生率が大幅に低下し、プロジェクトの安定性が向上する。<br>- **低:** 製造フェーズ完了後などに高確率で問題が発覚し、進捗が大幅に巻き戻るリスクを抱える。|
|**計算ロジック**|各フェーズ完了時に行われるイベント判定において、手戻りイベントの発生確率が以下の式で計算される。<br>`最終発生確率 = 基本発生確率 × (1 - (チームの平均設計値 × 0.05))`|

### 3.2 製造 (Develop)

プロジェクトの進捗スピードそのものを決定づける、最も基本的かつ直接的な生産能力です。このパラメータの合計値が、チームの基本的な馬力となります。（プレイヤーにとっては、最も目に見えて効果が分かりやすい「強さ」となります。）

|   |   |
|---|---|
|項目|詳細|
|**定義**|1ターン（1週間）あたりに消化できる**基本工数**を決定する能力。|
|**ゲーム内効果**|- **高:** ガントチャート上のタスクバーが目に見えて速く進む。<br>- **低:** 計画通りにタスクが消化できず、単純なスケジュール遅延の主原因となる。|
|**計算ロジック**|ターンごとの進捗計算式において、チーム全体の「製造」パラメータ合計値が、消化工数の基礎値として使用される。<br>`ターン進捗工数 = Σ(各担当者の製造値) × チーム状態補正`|

### 3.3 評価 (Test)

プロジェクトの最終的な品質を保証し、顧客満足度に直結する「最後の砦」です。（この能力の重要性を通じて、プレイヤーは「完成させること」と「品質を保証すること」の違いを学びます。）

|   |   |
|---|---|
|項目|詳細|
|**定義**|**「バグ発見率」**と、プロジェクト完了時の最終的な**「品質スコア」**を決定する能力。|
|**ゲーム内効果**|- **高:** テストフェーズで多くのバグを発見・修正でき、品質スコアが向上する。<br>- **低:** リリース後に致命的なバグが発覚し、顧客信頼度の急落や、莫大な補修コスト（ペナルティ）が発生するリスクがある。|
|**計算ロジック**|- テストフェーズで発見されるバグ数 `(基本発見率) * (1 + チームの平均評価値 * 0.05)`<br>- プロジェクト完了時の品質スコアボーナス `+(チームの平均評価値 * 2)`|

### 3.4 折衝 (Negotiation)

クライアントからの仕様変更要求や納期短縮といった、外部からの圧力からプロジェクトを守るための防御力です。（技術力だけでは乗り切れない局面、つまり「PMの仕事の9割はコミュニケーション」であることをプレイヤーに体験させます。）

|   |   |
|---|---|
|項目|詳細|
|**定義**|顧客からの**「仕様変更要求」**や**「納期短縮要求」**といった攻撃に対する防御力。|
|**ゲーム内効果**|- **高:** 交渉イベントにおいて、相手を説得して要求を退けたり、有利な条件（追加予算や納期延長）を引き出したりする選択肢の成功率が上がる。<br>- **低:** 顧客の要求を安易に受け入れてしまい、スコープクリープ（タスクの無限増殖）が発生しやすくなる。|
|**計算ロジック**|交渉イベント発生時、以下の対抗判定で成否を決定する。<br>`判定値 = (担当者の折衝値 × 補正係数) vs (相手の要求難易度)`|

### 3.5 提案 (Idea)

計画が行き詰まった際の状況を打開する「攻め」の能力です。技術的な課題やスケジュールの遅延といった困難な状況において、ブレイクスルーを生み出す閃きの力を示します。（プレイヤーに絶望的な状況からの逆転劇を提供する、ゲーム的なカタルシスを担うパラメータです。）

|   |   |
|---|---|
|項目|詳細|
|**定義**|プロジェクトの遅延や技術的課題を解決する、画期的なアイデアを閃く**「ブレイクスルーイベント」**の発生確率を示す能力。|
|**ゲーム内効果**|- **高:** 困難な状況下で、確率でブレイクスルーイベントが発生する。<br> - **イベント例:** 「既存ライブラリの活用を思いついた！」→ 特定タスクの工数が90%削減される。<br>- **低:** 問題解決を正攻法（リソース投入）に頼るしかなく、コストや時間が増大する。|
|**計算ロジック**|特定のネガティブイベント発生時、以下の確率でカウンターイベントが発生する。<br>`発生確率 = 基本発生確率 + (担当者の提案値 × 0.01)`|

### 3.6 判断 (Judgment)

無数の選択肢の中から「正解」を選び取り、チームの自律性を高めるPMの頭脳とも言える能力です。（この能力が高いメンバーは、PMのマイクロマネジメント負荷を下げてくれるため、プレイヤーはより戦略的な大局観に集中できます。）

|   |   |
|---|---|
|項目|詳細|
|**定義**|状況に応じてタスクの優先順位を正しく決定し、自律的に最適な行動を選択する能力。|
|**ゲーム内効果**|- **高:**<br> - PMが指示しなくても、クリティカルパス上のタスクを優先して処理する。<br> - トラブル発生時に、一次対応（ログ保全など）を自己完結させる。<br> - プレイヤーが選択を迫られるイベントで**「正解の選択肢」**にヒントが表示されることがある。<br>- **低:** 優先順位を無視したり、些細な問題で思考停止しPMに指示を仰いだりするため、管理コストが増大する。|
|**計算ロジック**|タスクの自動アサインやイベント選択肢のヒント表示は、以下の閾値判定で実行される。<br>`if (キャラクターの判断値 > 閾値) { AutoAction(); }`|

--------------------------------------------------------------------------------

【表】のスキルはプロジェクトを前に進めるための直接的な力ですが、それだけではチームはいつか疲弊し、崩壊します。次に、プロジェクトという長距離走を走り抜くための土台となる【裏】のパラメータについて解説します。

## 4.0 【裏】チーム運営スキルパラメータ

このセクションでは、直接的な生産活動には関与しないものの、チームというエンジンを円滑に動かし、プロジェクトの崩壊を防ぐために不可欠な6つの人間的・環境的パラメータを定義します。これらのパラメータは、プロジェクトの**「継戦能力」**や**「レジリエンス（回復・復元力）」**に大きな影響を与えます。

### 4.1 庶務 (Admin)

プロジェクトの直接的なタスク以外の雑務を処理し、PM（プレイヤー）の負担を軽減する、縁の下の力持ちとしての能力です。（このパラメータは、現実のプロジェクトを蝕む目に見えない組織的負荷を表現しており、高いキャラクターがいるとプレイヤーのQoLが向上したように感じられます。）

|   |   |
|---|---|
|項目|詳細|
|**定義**|経費精算、備品発注、会議室の予約といった、間接業務を処理する能力。|
|**ゲーム内効果**|- **高:** この能力を持つキャラクターがいるチームは、PMが毎ターン消費する**「行動ポイント（AP）」**や**「管理コスト」**が自動的に削減される。<br>- **低:** PMが雑務に追われ、本来注力すべきリスク管理や意思決定の時間が奪われる。|
|**計算ロジック**|ターン開始時のPMのAP計算において、コストが以下の式で削減される。<br>`最終AP消費量 = 基本消費量 - (チーム内の庶務合計値 × 0.5)`|

### 4.2 幹事 (Organizer)

チームの士気（モチベーション）を管理し、一体感を醸成するムードメーカーとしての能力です。（特にプロジェクトがデスマーチに陥った際の「V字回復」を演出するための重要なパラメータです。）

|   |   |
|---|---|
|項目|詳細|
|**定義**|チームの団結力を高め、低下した士気を回復させるイベントを発生させる能力。|
|**ゲーム内効果**|- **高:** 定期的に「飲み会」や「ランチ会」といった**士気回復イベント**が発生し、チーム全体の士気ゲージを大幅に回復させる。デスマーチ状態からの回復の鍵となる。<br>- **低:** チームの雰囲気は悪化する一方で、士気の自然回復が見込めない。|
|**計算ロジック**|各ターンの終了時、以下の確率で「士気回復イベント」の発生判定が行われる。<br>`発生確率 = 基本発生確率 + (チーム内の最大幹事値 × 0.02)`|

### 4.3 給仕 (Service)

メンバーの物理的なコンディションを維持し、高負荷な状況下での長期的な稼働を支える能力です。（お菓子を配る、コーヒーを淹れるといった細やかな気配りが、チームの継戦能力に繋がるというPMの現実を反映します。）

|   |   |
|---|---|
|項目|詳細|
|**定義**|メンバーの**「スタミナ（体力）」**の減少を緩和させる能力。|
|**ゲーム内効果**|- **高:** このキャラクターがいるチームは、全体のスタミナ減少率にマイナス補正がかかる。残業が続いてもメンバーが体調不良で離脱するリスクが低下する。<br>- **低:** メンバーがすぐに疲弊し、病欠による戦線離脱が多発する。|
|**計算ロジック**|ターン終了時の各メンバーのスタミナ計算において、減少量が以下の式で緩和される。<br>`最終スタミナ減少量 = 基本減少量 × (1 - (チーム内の給仕合計値 × 0.02))`|

### 4.4 雑談 (Chat)

メンバー間の人間関係の摩擦を解消する潤滑油であり、チーム内の対立を防ぐ「鎹（かすがい）」の役割を果たします。（スキルは高いが扱いにくいエース同士を共存させるための、高度なチームビルディングの面白さを提供します。）

|   |   |
|---|---|
|項目|詳細|
|**定義**|メンバー間の「相性」によるネガティブな影響を中和し、**対立イベントの発生**を防ぐ能力。|
|**ゲーム内効果**|- **高:** この能力を持つキャラクターを、性格的に相性の悪いメンバーの間に配置することで、「対立による進捗停止イベント」の発生を無効化できる。<br>- **低:** スキルは高いが相性の悪いメンバーで固めたチームは、内部対立により機能不全に陥る。|
|**計算ロジック**|相性の悪いメンバーが隣接している場合、「対立イベント」の発生確率が以下の式で抑制される。<br>`最終発生確率 = 基本発生確率 × (1 - (間にいる雑談値 × 0.1))`|

### 4.5 美貌 (Charm)

直接的な業務能力ではなく、その人間的魅力によって周囲のパフォーマンスに影響を与える特殊なパラメータです。（この能力による差別的なマイナス効果は一切なく、プラスの効果のみをもたらすことで、人間関係のポジティブな側面を表現します。）

|   |   |
|---|---|
|項目|詳細|
|**定義**|自分より美貌値が高い相手の前で「良いところを見せたい」という心理を誘発し、対象の能力に一時的なボーナスを与える能力。|
|**ゲーム内効果**|- **社内効果:** チーム内に美貌値が高いメンバーがいると、他のメンバーの「製造」や「提案」といった能力にプラス補正がかかる。<br>- **社外効果:** 顧客担当者よりこちらの担当者の美貌値が高い場合、商談の成功率やトラブル時の鎮静化にボーナスが付与される。|
|**計算ロジック**|**【社内効果】**<br>チーム内で最も美貌値が高いキャラクターをBとし、それ以外の各キャラクターをAとする。キャラクターAの「製造」「提案」値に、以下のボーナスが加算される。<br>ボーナス値 `= (Bの美貌値) - (Aの美貌値)`<br>※この計算結果が0以下の場合、ボーナスは発生しない。キャラクターB自身にはボーナスは発生しない。<br><br>**【社外効果】**<br>顧客担当者をB、こちらの担当者をAとする。交渉成功率などに、以下のボーナスが加算される。<br>ボーナス値 `= (Aの美貌値) - (Bの美貌値)`<br>※この計算結果が0以下の場合、ボーナスは発生しない。|

### 4.6 幸運 (Luck)

論理や実力では説明できない奇跡を引き起こす、予測不能な変数です。（プレイヤーの実力だけではどうにもならない状況を打開する「理不尽な救済措置」として機能し、ゲームのドラマ性を高めます。）

|   |   |
|---|---|
|項目|詳細|
|**定義**|通常時は何の効果もないが、ごく低確率でプロジェクトの危機を救う**「ラッキーイベント」**を引き起こす能力。|
|**ゲーム内効果**|発動時には、以下のような理不尽なほどのプラスイベントが発生する。<br>- クライアントの都合による仕様変更の**撤回**<br>- 原因不明のクリティカルバグが**自然治癒**<br>- 競合他社の自滅による入札の**不戦勝**|
|**計算ロジック**|ターン毎に以下の確率でイベントが発生する。「起死回生」仕様として、プロジェクトの危機的状況に応じて`危機補正`が乗算される。<br>`発生確率 = (チーム内の幸運合計値 × 0.005) × 危機補正`|

--------------------------------------------------------------------------------

これらのパラメータは、キャラクターの個性そのものです。では、これらの能力はどのように変化していくのでしょうか。次章では、キャラクターの成長に関するルールを定めます。

## 5.0 パラメータの成長ルール

キャラクターの成長に関する基本方針は、「スキルは後天的に習得できるが、人間性は天賦の才である」という思想に基づいています。これにより、育成と採用の両方に戦略的な意味を持たせます。

- **【表】プロジェクトスキルの成長:** プロジェクトへの参加経験や、研修イベントによって経験値が蓄積され、最大値（10）まで成長します。プレイヤーは、若手を育成してエースに育てる楽しみを味わうことができます。
- **【裏】チーム運営スキルの成長:** 原則として、ゲーム開始時の初期値から**成長しません**（あるいは非常に成長しにくい）。これは、そのキャラクターが生まれ持った人間性や適性という扱いのためです。この仕様により、裏の能力が高い希少な人材を発掘・採用（スカウト）する重要性が、ゲームの終盤まで維持されます。
- **例外:** 特定の「人生の転機」といった非常にレアなイベントでのみ、【裏】パラメータが予期せず変動する可能性があります。

## 6.0 補足：パラメータのランダム生成

「二つ名」を持つ固有NPC以外の一般キャラクターが生成される際の、パラメータ割り振りの基本ルールを定めます。これは、ステレオタイプを排除し、プレイごとに異なる出会いとチーム編成の悩みを提供することで、リプレイ性を高めるための設計です。

- **ポイント総量制:** キャラクターのランク（新人、中堅、ベテランなど）に応じて、割り振ることができるパラメータポイントの総量のみが決定されます。
- **完全ランダム配分:** 決定されたポイント総量を、**【表】【裏】合わせた全12パラメータに、完全にランダムに配分**します。「エンジニアだから製造が高い」といった職種による補正は一切かけません。
- **生成例:** このルールにより、現実世界のように多様で予測不能な人材が生まれます。
    - 「製造は1だが美貌が10のエンジニア」
    - 「設計も折衝も壊滅的だが、幸運だけが突出している事務員」
    - 「全ての能力が平均的に5で、可もなく不可もない凡人」

プレイヤーは、毎回異なる個性を持つメンバーたちと向き合い、その凸凹な能力をどう組み合わせ、プロジェクトを成功に導くかという、マネジメントの根源的な面白さに挑戦することになります。
