# 新規ゲーム企画『PM立志伝：プロジェクト・クロニクル』プロジェクト提案書

## 1. 企画概要 (Project Overview)

本企画は、単なる経営シミュレーションゲームではありません。これは、現代ビジネス必須の専門スキルであるプロジェクトマネジメント（PM）を、ゲーミフィケーションを通じて体得させる、次世代の企業研修を定義する「デジタル道場」の提案です。プレイヤーはプロジェクトマネージャーとして、予算、納期、そして複雑な人間関係という制約の中で価値を創造する挑戦に没入します。成功と失敗の体験を高速で繰り返すことで、教科書では得られない生きたPMスキルを習得する、全く新しい学習アプリケーションを創造します。

- **タイトル案:** 『PM立志伝：プロジェクト・クロニクル』
- **キャッチコピー:** ～デスマーチを回避し、伝説のPMを目指せ～
- **ジャンル:** プロジェクトマネジement体感シミュレーション / 学習アプリ
- **コンセプト:** カイロソフト社の『ゲーム発展国++』が持つ中毒性のある経営・育成ループと、コーエーテクモゲームス社の『太閤立志伝V』が持つ自由度の高いキャリア・スキル習得システムを融合させ、PMの実務を体感的に学習する。
- **コア原則:** 運要素を排し、「PMBOK（プロジェクトマネジメント知識体系）」に準拠した正しい知識と戦略的な判断が成功の鍵となる、論理的で再現性の高いゲーム設計。

本企画は、純粋なエンターテインメント性と実用的な教育価値を両立させることで、 edutainment（エデュテインメント）という新たな市場セグメントを確立します。次章では、本企画が狙う巨大な市場機会と、その中核をなすターゲット層について詳述します。

## 2. 市場背景とターゲット層 (Market Context and Target Audience)

現代のあらゆるビジネスにおいて、プロジェクトマネジメント能力は職種を問わず必須のスキルです。しかし市場は、知識を一方的に伝達する教科書やビデオコースといった「受動的な学習教材」で飽和しており、真の実践能力を育成するという点では決定的なギャップが存在します。本企画は、PMが「失敗しても安全な環境」で、結果が伴う経験を通じてスキルを鍛えたいという、巨大かつ満たされていない需要をターゲットとします。

本ゲームは、以下の3つの主要なユーザー層をターゲットとしています。

|   |   |   |
|---|---|---|
|ターゲット層|特徴|本ゲームから得られる価値|
|**1. 未来のPM層（学生・若手社会人）**|これからPMやリーダーを目指す、あるいはキャリアに漠然とした不安を抱える層。実務経験が少ない。|プロジェクト全体の流れと、PMが日々直面する課題を疑似体験できる。専門用語や思考プロセスを、キャリアの早い段階で楽しく学べる。|
|**2. 現役のPM・リーダー層**|すでにPMとして活動しているが、自己流のマネジメントに限界を感じている、あるいはより大規模な案件に挑戦したいと考えている層。|自身のマネジメントスタイルを客観的に見直し、PMBOKに基づいた知識との答え合わせができる。現実では試せない大胆な戦略をシミュレーションできる。|
|**3. マネジメントに関心のある一般ビジネスパーソン**|PM職ではないが、チームでの業務遂行やリーダーシップに関心を持つ全てのビジネスパーソン。|「なぜプロジェクトは炎上するのか」「なぜ上司はあのような判断をするのか」といった、組織力学の裏側を理解できる。自身の業務調整能力向上に繋がる。|

これらのターゲット層に共通するのは、「体系的かつ実践的なPMスキルを、座学ではなく体験を通じて学びたい」という強い動機です。本企画は、このニーズに真正面から応えるソリューションとして設計されています。

## 3. コアコンセプトと主要メカニクス (Core Concept and Key Mechanics)

本ゲームの根幹をなす設計思想は、PMが直面する最大の挑戦、すなわち「混沌と制約の中で価値を創造する」という、ダイナミックな戦略シミュレーションを提供することにあります。これから解説する各メカニクスは、それぞれが独立して存在するのではなく、相互に作用し、プレイヤーに常にトレードオフの決断を迫るよう緻密に連携しています。

### ゲームサイクル：1ターン＝1週間のPDCA

ゲームの基本進行は「1ターン＝1週間」とし、アジャイル開発のスプリントや週次定例報告のリズムを再現します。これにより、プレイヤーはマイクロマネジメントではなく、週次の傾向を掴んで次の一手を考えるという、PM本来の思考を追体験できます。

1. **週初めの計画（Monday Plan）** 今週の目標達成に向け、リソース配分と方針を決定する戦略フェーズです。プレイヤーはガントチャートを見ながら、どのタスクに誰をアサインするかを決定します。さらに「品質第一」「突貫工事」「定時退社」といった方針カードを選択することで、チーム全体の作業効率やリスクに影響を与えます。
2. **平日の実行とイベント（Tue - Thu Run）** 計画フェーズを終えると、火曜日から木曜日までは計画に基づき自動で進行します。この間、プレイヤーのアサインやメンバーの能力に応じて「仕様の曖昧な点を発見した」「顧客から無理な要求が来た」といった様々なイベントが発生し、プレイヤーはその都度、判断を求められます。
3. **週末の振り返り（Friday Review）** 金曜日の夕方には週次レポートが表示されます。EVM（アーンド・バリュー・マネジメント）指標であるSPI（スケジュール効率指数）とCPI（コスト効率指数）によって、計画に対する進捗とコストの状況が可視化されます。また、各メンバーの体力やモチベーションも確認し、次週の計画に活かします。

### キャラクターシステム：個性の集合体が織りなすドラマ

本ゲームの最大の特徴は、単なる「労働力」ではない、人間味あふれるキャラクターたちです。彼らの能力、性格、価値観がプロジェクトの成否を大きく左右します。

- **能力値『ダブル・ヘキサゴン』** キャラクターの能力は、業務スキルを示す「表の六角形」と、人間性や適性を示す「裏の六角形」の合計12項目で表現されます。これにより、「技術は超一流だがチームを破壊する天才」や「スキルは低いがムードメーカーとして不可欠な潤滑油」といった、多面的でリアルな人物像を描き出します。
    - **表の能力:** 設計, 製造, 評価, 折衝, 提案, 判断
    - **裏の能力:** 庶務, 幹事, 給仕, 雑談, 美貌, 幸運
- **固有NPCと『二つ名』システム** 物語に深く関わる固有NPCが存在します。彼らは当初、一癖も二癖もある人物として登場しますが、プレイヤーがPMとして彼らの本質を見抜き、信頼関係を築くことで真の能力が解放され、『深夜のコミット怪人』『微笑みの仕様粉砕者』といった「二つ名」と共に、プロジェクトの切り札となります。
- **一般キャラクターのランダム性と『覚醒』** ランダム生成される一般キャラクターにも、「二つ名持ち」「実務の鬼（二つ名はないがステータスが急上昇）」「一芸特化」といった「覚醒タイプ」が秘められています。これにより、プレイヤーは履歴書の片隅に書かれたヒントから「埋もれた原石」を見つけ出し、育成するという、リプレイ性の高い人材発掘の楽しみを味わえます。
- **体力システム『瞬発型と持続型』** 体力は「最大体力（タンクの大きさ）」と「回復力（回復スピード）」の2軸で定義されます。これにより、「無理は効かないが回復が早い若手（瞬発型）」と、「連続稼働はできるが一度倒れると復帰が遅いベテラン（持続型）」という運用特性が生まれます。PMは、メンバーを使い潰さないための高度なリソース管理能力が求められます。

最終的に、このキャラクターシステムはPMの複合的な挑戦を再現します。『ダブル・ヘキサゴン』が個々のポテンシャルを定義し、『二つ名』と『覚醒』が発見と成長の旅路を創出し、そして『体力システム』が現実的な運用の制約を課します。優れたPMは、ポテンシャルの発掘、成長の促進、そして疲労の管理という三つの側面すべてをマスターし、勝利するチームを構築しなければなりません。

### 交渉カードバトル：PM用語が武器になる

ステークホルダー（顧客、上司、メンバー）との重要な対話は、カードバトル形式で表現されます。「スコープ・クリープ」「トレードオフ」「エスカレーション」といったPMBOKの専門用語が、攻撃、防御、特殊効果を持つカードとして登場します。キャラクターの「折衝」能力値が初期手札の枚数や特定カードの威力に影響するなど、人材育成が交渉の成否に直結します。これにより、プレイヤーは楽しみながら、自然と専門用語とその実践的な使い方を習得できます。

### カレンダーシステム：季節という名の不可避な制約

ゲームは4月から始まる12ヶ月制で進行します。ゴールデンウィーク、夏季休暇、年末年始といった長期休暇は、プロジェクトの進捗を阻む「鬼門イベント」として固定配置されています。キャラクターごとに設定された『休暇スタンス』（バカンス勢、出稼ぎ勢など）を考慮し、休暇前に『シフト調整パズル』を解かなければなりません。これは、PMが直面する現実的な人間関係調整の苦悩と面白さを再現します。

これらのメカニクスが有機的に絡み合うことで、プレイヤーは常に「限られたリソースの中で最適解を導き出す」という、プロジェクトマネジメントの本質的な挑戦と向き合うことになります。

## 4. 学習要素と教育的価値 (Learning Elements and Educational Value)

本企画は単なるゲームに留まらず、プレイヤーが楽しみながら実践的なPMスキルを習得できる、効果的な学習ツールとして設計されています。ゲームをクリアする頃には、プレイヤーがPMとしてのマインドセットとスキルセットを自然に体得している状態を目指します。

### 学習サイクル：実践と振り返りの高速ループ

本ゲームは、以下の4ステップから成る学習サイクルを高速で繰り返すことで、知識の定着を促します。

1. **Play (実践)** PMとして、プロジェクトの様々な局面で判断を下します。成功もあれば、手痛い失敗もあります。
2. **Speak (言語化)** 交渉カードバトルにおける『直談判』カードなど、重要な局面でプレイヤーは自分の言葉で相手を説得することを求められます。このステップは、単に選択肢を選ぶ行為から脱却し、自らの戦略的意図を「言語化」する能動的なプロセスを促します。思考を言葉にする訓練は、学習内容を深く定着させ、優れたPMの証である真の説得力を養うための強力なツールです。
3. **Review (振り返り)** プロジェクト完了後、後述する生成AIによるレポートを通じて、自らの行動がPM理論においてどのような意味を持っていたのかを客観的に振り返り、答え合わせをします。
4. **Learn (習得)** レポートで示された専門用語や改善点を深く理解し、次のプロジェクトで実践することで、学んだ知識を確かなスキルへと昇華させます。

### 生成AIによるパーソナライズされたフィードバック

本企画の最も革新的な機能が、生成AIを活用した「プロジェクト完了報告書」です。ゲーム中のプレイヤーの全行動ログをAIが解析し、「ベテランPMメンター」のペルソナで、一人ひとりに最適化された詳細なフィードバックを提供します。

- **【Your PM Style】あなたの行動特性** プレイヤーの意思決定ログから、「リスク回避型」「コスト重視型」といったPMとしての行動傾向を分析・言語化します。これにより、プレイヤーは自身の強みと弱みを客観的に把握できます。
- **【Critical Decisions】運命の分岐点** プロジェクトの成否を分けた重要な局面をピックアップし、プレイヤーの判断がなぜ正解（または不正解）だったのかを、PMBOKの理論を交えて具体的に解説します。
- **【Vocabulary Note】今回学んだ用語帳** ゲーム中で遭遇した「WBS」「クリティカルパス」「ステークホルダー」といった専門用語が、プレイヤー自身の行動と紐付けられた形でリスト化されます。「用語を覚える」のではなく「用語を使った」経験として記憶に残ります。
- **【KPT】次へのアクション** Keep（良かったこと）、Problem（悪かったこと）、Try（次に試すべきこと）のフレームワークで、具体的な改善アクションを提示します。これにより、プレイヤーは次のプレイへの明確な目標を持つことができます。

このように、実践と個別フィードバックを高速で繰り返す「体験学習ループ」こそが、従来の教科書的な学習では決して得られない「生きたPMスキル」をプレイヤーに授けるのです。

## 5. ゲームフローとUI設計 (Game Flow and UI Design)

本ゲームが内包する複雑なシステムを、プレイヤーにストレスなく、かつ直感的に理解させるためには、洗練されたUI/UX設計が不可欠です。本企画では、PMが日々向き合う多様な情報を一元管理する「戦況図（コックピット）」をコンセプトに、視認性と情報密度を両立させた画面設計を目指します。

### ゲーム開始フロー：世界に一つだけの舞台設定

1. **主人公メイキング** 最初に、プレイヤー自身の分身となる主人公を作成します。「年齢設定（20代/30-40代/50代〜）」によって、体力や初期スキルといった基礎ステータスが決定されます。その後、いくつかの質問に答える「PMタイプ診断」を通じて、得意分野がカスタマイズされ、プレイヤーごとに異なるプレイスタイルが生まれます。
2. **ワールドジェネレーション** ゲーム開始時に自社、競合他社、市場案件などのエコシステムを固定生成することは、本作の譲れない設計思想の柱です。これにより、「後出しジャンケン」のような理不尽なランダム性が排除され、成功が予測不可能なイベントへの反応ではなく、先見性と戦略的計画によって決定される公平な競争環境が保証されます。全てのプレイが、ユニークかつ公正な戦略的パズルとなるのです。

### メイン画面『PMコックピット』：戦況の一覧化

ゲームプレイの大半を過ごすことになるメイン画面は、以下の4つのエリアで構成され、プロジェクトの全体像を一目で把握できるよう設計されています。

- **上部: プロジェクト・ステータスバー** カレンダー（現在の週、季節）、EVMメーター（SPI/CPIの針がリアルタイムで変動）、残り予算など、プロジェクトの全体的な健康状態を示す計器類が表示されます。
- **中央左: チーム・リソースマップ** メンバーの状態を、単なるリストではなくオフィスの座席表のようなビジュアルで表現します。各キャラクターの頭上には体力ゲージが表示され、誰が疲弊しているか、誰が不満を抱えているかを直感的に把握できます。
- **中央右: ガントチャート・ボード** プロジェクト進捗の心臓部です。グレーの計画線の上を、青い実績線が伸びていくことで、計画との乖離が一目瞭然となります。遅延しているタスクは赤く発光し、プレイヤーに警告を発します。
- **下部: コマンドデッキ** 週の初めにタスクをアサインしたり、チーム全体の方針を指示したりと、プレイヤーが具体的なアクションを行うためのエリアです。

このコックピット設計は、プレイヤーがデータ検索に忙殺されることを防ぎます。その代わり、戦略的な状況全体を一目で把握し、真に重要なこと、すなわち迅速かつ質の高い「判断」を下すことに集中できるよう、プレイヤーを力強く支援します。

## 6. 開発ロードマップ（案） (Proposed Development Roadmap)

本企画のような野心的なプロジェクトを成功裏に収めるためには、一度に全てを実装しようとするのではなく、段階的かつ現実的な開発アプローチが不可欠です。まず、ゲームの核となる面白さを証明するMVP（Minimum Viable Product: 実用最小限の製品）を構築し、そこから着実に機能とコンテンツを拡張していく方針を提案します。

|   |   |   |
|---|---|---|
|フェーズ|目標|主な実装機能|
|**フェーズ1: MVP (Minimum Viable Product)**|**コア・ループの証明:** 週次の計画・実行・振り返りのサイクルが、根本的に面白く、繰り返しプレイ可能であることを確立する。|・週次PDCAサイクル（計画→実行→振り返り）の基本ロジック<br>・ダブルヘキサゴンによるキャラクター能力値システム<br>・シンプルな交渉カードバトルとEVMによる評価システム<br>・固有NPCの「二つ名」は、信頼度によるステータス上昇のみ実装|
|**フェーズ2: フレーバーアップデート**|**没入感の深化:** 世界観を豊かにし、キャラクターへの愛着を醸成する演出を強化する。|・12ヶ月カレンダーと季節イベント（GW、夏季休暇など）の導入<br>・AIによる「プロジェクト完了報告書」のプロトタイプ実装<br>・固有NPCの「二つ名」解放時のカットイン演出やテキスト追加<br>・UIのビジュアルブラッシュアップ|
|**フェーズ3: イベントアップデート**|**戦略的深度の拡張:** 複雑なイベントとキャラクター固有能力を実装し、リプレイ性と高度な戦略性をプレイヤーに提供する。|・入札バトルシステム、公的機関案件などの特殊プロジェクト<br>・キャラクターの特性（Trait）に紐づいた固有イベントの実装<br>・『直談判』カードなどの自由記述システムの導入<br>・生成AIレポートの機能拡充と精度向上|

このロードマップに基づき、各フェーズで明確な目標を設定し、リスクを管理しながら着実にプロジェクトを推進することで、高品質な製品を計画通りに市場へ届けることが可能であると確信しています。

## 7. まとめ (Conclusion)

本企画『PM立志伝：プロジェクト・クロニクル』は、単なるシミュレーションゲームの枠を遥かに超え、次世代のプロジェクトマネージャーを育成するための革新的な教育プラットフォームとして、企業研修の新たな業界標準を打ち立てます。これは、エンターテインメントと教育が融合した「デジタル道場」であり、プレイヤーは熱中する中で、現実のビジネスで通用する思考法、判断基準、そして人間関係構築スキルを体得していくでしょう。
