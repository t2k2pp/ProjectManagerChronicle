# React コードレビュースキル

## 概要

このスキルは、React/TypeScript プロジェクトのソースコードを体系的にレビューし、コードの品質、保守性、パフォーマンス、型安全性に関する問題を特定します。

## トリガー

以下のような依頼でこのスキルを使用してください：

- 「このReactコードをレビューして」
- 「コンポーネントのコードレビューをお願い」
- 「React のコードの問題点を見つけて」
- `/react-review` コマンド

## レビューモード

### クイックレビュー（デフォルト）

対象：PRレビュー、単一ファイル、5-10分程度
チェックリスト：[quick-review.md](./checklists/quick-review.md)

### フルレビュー

対象：詳細分析、アーキテクチャ評価、30分以上
チェックリスト：[full-review.md](./checklists/full-review.md)

## レビュー手順

1. **対象の特定**: レビュー対象のファイル/ディレクトリを確認
2. **モードの選択**: クイックまたはフルレビューを選択
3. **チェックリストの実行**: 該当するチェックリストに従って確認
4. **結果の出力**: [review-report.md](./templates/review-report.md) テンプレートで報告

## レビュー観点

### 1. コード品質・可読性
- 命名規則（camelCase、PascalCase）
- フォーマット・スタイルの一貫性
- DRY原則（重複コードの排除）
- 複雑性（単一責任原則）
- コメント・ドキュメント

### 2. React 固有のベストプラクティス
- **コンポーネント設計**
  - 適切なサイズ（200-300行以下推奨）
  - ロジックとプレゼンテーションの分離
  - 再利用可能性
- **Hooks の使用**
  - Hooks のルール遵守（トップレベルでのみ呼び出し）
  - useEffect の依存配列の正確性
  - useCallback/useMemo の適切な使用
- **Props と State**
  - Props の型定義
  - State の適切な管理
  - key prop の正しい使用

### 3. TypeScript 型安全性
- `any` 型の回避
- 明示的な型定義
- ユーティリティ型の活用（Pick, Omit, Partial など）
- ジェネリクスの適切な使用

### 4. パフォーマンス
- 不要な再レンダリングの回避
- メモ化（React.memo, useMemo, useCallback）
- コード分割・遅延ロード

### 5. テスト
- ユニットテストの存在
- 重要なロジックのカバレッジ
- テスト可能な設計

### 6. アクセシビリティ
- セマンティック HTML
- ARIA 属性
- キーボードナビゲーション

### 7. セキュリティ
- 入力検証
- XSS 対策
- 機密情報の取り扱い

## 重要度分類

レビュー結果は以下の重要度で分類します：

| レベル | 説明 | 対応 |
|-------|------|------|
| 🚨 Critical | バグ、セキュリティ問題 | 必須修正 |
| ⚠️ Warning | パフォーマンス、設計問題 | 推奨修正 |
| 💡 Suggestion | ベストプラクティス、改善提案 | 検討 |
| ℹ️ Info | 情報提供、参考事項 | 任意 |

## 出力フォーマット

レビュー結果は以下のセクションで構成します：

1. **サマリー**: 全体評価と主要な発見
2. **Critical/Warning**: 必須/推奨の修正項目
3. **Suggestions**: 改善提案
4. **コード例**: 具体的な修正例（必要に応じて）
