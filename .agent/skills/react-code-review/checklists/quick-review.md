# クイックレビュー チェックリスト

PRレビューや単一ファイルの簡易チェック用（5-10分）

## 必須チェック項目

### コード品質
- [ ] console.log / debugger 文が残っていないか
- [ ] コメントアウトされたコードが残っていないか
- [ ] 未使用の import / 変数がないか
- [ ] ハードコードされた値（マジックナンバー、URL等）がないか
- [ ] 命名が明確で一貫性があるか

### React 基本ルール
- [ ] key prop が正しく設定されているか（配列のindex使用を避ける）
- [ ] useEffect の依存配列が正しいか
- [ ] Hooks がコンポーネントのトップレベルで呼ばれているか
- [ ] Props に適切な型定義があるか
- [ ] コンポーネントが大きすぎないか（200行以下推奨）

### TypeScript
- [ ] `any` 型が使用されていないか
- [ ] 型エラーがないか
- [ ] 必要な型定義が存在するか

### 明らかな問題
- [ ] 無限ループの可能性がないか
- [ ] メモリリークの可能性がないか（未クリーンアップの副作用）
- [ ] エラーハンドリングが適切か
- [ ] ユーザー入力の検証があるか

## チェック方法

1. ESLint/TypeScript の警告・エラーを確認
2. 変更差分を上から順にレビュー
3. 上記チェックリストを確認
4. 不明点は質問またはコメント
