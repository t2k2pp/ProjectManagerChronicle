# 設計書の品質管理指標

## 概要

設計書の品質を定量的に評価するための指標です。

---

## 1. 完全性指標

### 要件カバレッジ率
```
カバレッジ率 = (設計に反映された要件数 / 全要件数) × 100%
```
- 目標値: 100%
- 許容値: 95%以上（ただし未カバー理由が明確なこと）

### 未解決項目率
```
未解決率 = (TBD/TODO項目数 / 全設計項目数) × 100%
```
- 目標値: 0%（レビュー完了時点）
- 許容値: 5%以下（期限と担当者が明確なこと）

---

## 2. 具体性指標

### 曖昧表現率
以下のキーワードがある項目の割合：
- 「適切に」「必要に応じて」「状況によって」
- 「など」「等」で終わる列挙
- 数値がない「高速」「大量」「十分な」

```
曖昧率 = (曖昧表現を含む項目数 / 全項目数) × 100%
```
- 目標値: 0%
- 許容値: 5%以下

### 数値定義率
数値で定義すべき項目（タイムアウト、サイズ制限、件数等）のうち、
具体的な数値が定義されている割合。

```
数値定義率 = (具体的数値がある項目数 / 数値定義すべき項目数) × 100%
```
- 目標値: 100%
- 許容値: 90%以上

---

## 3. 整合性指標

### 用語一貫性
同一概念に対して異なる用語が使用されている箇所の数。

- 目標値: 0箇所
- 許容値: 0箇所（用語は必ず統一）

### 設計書間整合率
関連する設計書との矛盾がない割合。

```
整合率 = (整合している参照数 / 全参照数) × 100%
```
- 目標値: 100%
- 許容値: 100%（矛盾は許容しない）

---

## 4. レビュー指標

### レビュー指摘密度
設計書のページ数あたりの指摘件数。

```
指摘密度 = 指摘件数 / 設計書ページ数
```
- 目安: 5件/ページ以上（品質確保のため）
- 指摘が少なすぎる場合はレビューの質を疑う

### Critical/Warning 修正率
```
修正率 = (修正完了した Critical + Warning 件数) / (全 Critical + Warning 件数) × 100%
```
- 目標値: 100%（次工程移行条件）
- Critical は必ず 100%修正

---

## 5. 品質評価基準

### 総合評価

| 評価 | 要件カバレッジ | 未解決率 | 曖昧率 | 数値定義率 | Critical修正率 |
|------|--------------|---------|--------|----------|---------------|
| A（優秀） | 100% | 0% | 0% | 100% | 100% |
| B（良好） | 95%以上 | 3%以下 | 3%以下 | 95%以上 | 100% |
| C（許容） | 90%以上 | 5%以下 | 5%以下 | 90%以上 | 100% |
| D（要改善） | 上記未満 | - | - | - | - |

### 次工程移行基準
- 評価 C 以上
- Critical 修正率 100%
- Warning 修正率 80%以上（残りは理由明確で次工程で対応予定）
