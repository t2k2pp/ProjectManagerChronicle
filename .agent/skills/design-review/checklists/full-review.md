# フルレビュー チェックリスト

複数設計書の網羅的なレビュー用（30分以上）

---

## 1. 要件との整合性

### 機能要件
- [ ] 要件定義書のすべての機能が設計に含まれているか
- [ ] 機能の優先度に応じた設計レベルになっているか
- [ ] 要件変更が設計に反映されているか
- [ ] 要件と設計の間にトレーサビリティがあるか

### 非機能要件
- [ ] 性能要件（応答時間、スループット、同時接続数）が設計に反映されているか
- [ ] 可用性要件（稼働率、冗長化、障害復旧）が考慮されているか
- [ ] セキュリティ要件が設計に落とし込まれているか
- [ ] 運用・保守要件（バックアップ、監視、ログ）が含まれているか
- [ ] スケーラビリティ要件が考慮されているか

## チェック方法

1. **事前準備**: 関連する要件定義書、他の設計書を準備
2. **構造確認**: 目次・章構成を確認し、必要な項目が網羅されているか確認
3. **トレーサビリティ確認**: 要件→設計の対応を確認
4. **キーワード検索**: TBD/TODO/検討中/適切に/必要に応じて を検索
5. **設計書間の整合性**: 他の設計書との差異・矛盾を確認
6. **上記チェックリストを順番に確認**
7. **発見事項の重要度を分類してレポート作成**
