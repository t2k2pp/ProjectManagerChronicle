# クイックレビュー チェックリスト

単一設計書の簡易チェック用（10-15分）

## 必須チェック項目

### 完全性

- [ ] 要件定義書の機能がすべて設計に反映されているか
- [ ] 画面設計・API設計・DB設計など必要な要素が揃っているか
- [ ] 未定義項目（TBD、TODO、検討中）が残っていないか
- [ ] 「詳細は別途定義」「後続工程で決定」などの先送り事項がないか

### 具体性

- [ ] 数値で定義すべき項目に具体的な値があるか（タイムアウト、件数制限等）
- [ ] 「適切に」「必要に応じて」など曖昧な表現がないか
- [ ] エラー発生時の具体的な処理・メッセージが定義されているか
- [ ] 判断基準・条件分岐の条件が明確か

### 一貫性

- [ ] 用語が統一されているか（同じ概念に異なる名前がないか）
- [ ] ID・コード体系が一貫しているか
- [ ] 他の設計書と矛盾する記述がないか

### 明らかな問題

- [ ] 存在しない画面・API・テーブルへの参照がないか
- [ ] 循環参照や無限ループの可能性がないか
- [ ] セキュリティ上の明らかな問題がないか（パスワード平文保存等）
- [ ] 現実的でない要件（無限のストレージ、ゼロ応答時間等）がないか

## AI生成設計チェック

- [ ] 「など」「等」で終わる列挙が完全なリストになっているか
- [ ] 汎用的すぎる説明がないか（具体例がない抽象的な記述）
- [ ] 同じフレーズのコピペパターンがないか
- [ ] エッジケース・異常系が考慮されているか

## チェック方法

1. 目次・章構成を確認し、必要な項目が網羅されているか確認
2. TBD/TODO/検討中をキーワード検索
3. 曖昧表現（「適切に」「必要に応じて」「状況によって」）を検索
4. 数値定義が必要な箇所（サイズ、時間、件数）を確認
5. 上記チェックリストを確認
