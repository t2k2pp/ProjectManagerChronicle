# 設計書テンプレート例

設計書作成時に含めるべき項目のリファレンス

---

## 基本設計書テンプレート

### 1. 概要
- 1.1 目的
- 1.2 対象範囲
- 1.3 関連ドキュメント
- 1.4 用語定義

### 2. システム概要
- 2.1 システム構成図
- 2.2 機能一覧
- 2.3 外部システム連携

### 3. 機能設計
- 3.1 機能詳細
  - 概要
  - 入力
  - 処理
  - 出力
  - 異常系

### 4. 非機能要件設計
- 4.1 性能要件
- 4.2 可用性要件
- 4.3 セキュリティ要件
- 4.4 運用・保守要件

### 5. データ設計
- 5.1 ER図
- 5.2 テーブル一覧
- 5.3 データフロー

### 6. インターフェース設計
- 6.1 外部インターフェース
- 6.2 内部インターフェース

### 7. 制約事項・前提条件

### 8. 変更履歴

## 項目記述のガイドライン

### 必ず具体的に記述する項目
- **数値**: タイムアウト（秒）、サイズ制限（バイト、MB）、件数制限
- **コード**: エラーコード体系、ステータスコード
- **メッセージ**: エラーメッセージ、確認メッセージ
- **条件**: 分岐条件、判断基準、閾値

### 避けるべき表現
| 避けるべき | 代わりに |
|-----------|---------|
| 適切に処理する | 具体的な処理手順を記述 |
| 必要に応じて | いつ必要かの条件を明記 |
| など | 完全なリストを列挙 |
| 高速に | 具体的な応答時間（3秒以内等） |
| 大量のデータ | 具体的な件数（10万件等） |

### 異常系の記述例
```
エラー発生時:
- エラーコード: ERR-AUTH-001
- エラーメッセージ: 「認証に失敗しました。ユーザーIDまたはパスワードを確認してください。」
- 画面遷移: ログイン画面に戻る
- ログ出力: WARNレベルでユーザーID（マスク済み）と失敗理由を出力
- リトライ: 3回まで許可、4回目で15分間ロック
```
