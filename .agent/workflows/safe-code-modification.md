# コード修正ワークフロー

## 禁止事項
1. **PowerShellやシェルでの一括置換は禁止**
   - 日本語ファイルの文字化けリスクが高い
   - エンコーディング問題で大量のエラーが発生する

2. **複数ファイルの同時一括編集は禁止**
   - 問題発生時の切り分けが困難になる

## 推奨手順
1. **1ファイルずつ**編集ツール（replace_file_content/multi_replace_file_content）で修正
2. **修正箇所を確認**してから次のファイルへ
3. **定期的にビルド確認**（5ファイル程度ごと）
4. **コミットを細かく**行い、問題発生時にロールバック可能にする

## 大規模リファクタリング時
1. 計画を立てる（対象ファイル一覧作成）
2. フェーズに分けて実行
3. 各フェーズ完了後にコミット
4. 急がば回れ - 丁寧に進める
